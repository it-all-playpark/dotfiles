## Claude Code

### /powerup インタラクティブレッスン機能
- **ソース**: [GitHub CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)
- **要約**: Claude Code の機能をアニメーション付きデモで学べるインタラクティブレッスン `/powerup` が追加された。新規ユーザーのオンボーディングや、既存ユーザーが知らない機能の発見に役立つ。
- **影響**: 新しいチームメンバーへの Claude Code 導入が容易になる。

### PreToolUse hook に "defer" パーミッション決定を追加 (v2.1.89)
- **ソース**: [ClaudeCodeLog on X](https://x.com/ClaudeCodeLog/status/2039153164717334706), [Hooks Reference](https://code.claude.com/docs/en/hooks)
- **要約**: PreToolUse hook が `permissionDecision: "defer"` を返せるようになった。ヘッドレスセッションでツールコール時に一時停止し、`-p --resume` で再開してフックを再評価できる。CI/CD パイプラインでの人間承認ゲートに有用。
- **影響**: ヘッドレス運用での柔軟なパーミッション制御が可能に。本番環境へのデプロイ承認フローなどに活用できる。

### PermissionDenied hook の追加
- **ソース**: [Hooks Reference](https://code.claude.com/docs/en/hooks), [Pixelmojo](https://www.pixelmojo.io/blogs/claude-code-hooks-production-quality-ci-cd-patterns)
- **要約**: auto mode でクラシファイアがツールコールを拒否した際に発火する新しいフックイベント。`retry: true` を返すことでリトライを許可できる。手動拒否や PreToolUse hook によるブロックでは発火しない。
- **影響**: auto mode の挙動を詳細にログ・制御できるようになり、意図しない拒否への対処が自動化可能。

### CLAUDE_CODE_NO_FLICKER 環境変数
- **ソース**: [ClaudeCodeLog on X](https://x.com/ClaudeCodeLog/status/2039153164717334706)
- **要約**: `CLAUDE_CODE_NO_FLICKER=1` を設定するとフリッカーフリーレンダリングが有効になり、ターミナル表示のちらつきが解消される。
- **影響**: ターミナル録画やデモ時に特に有用。視覚的な快適性が向上。

### Edit ツールの最適化
- **ソース**: [GitHub CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)
- **要約**: Edit ツールが短い `old_string` アンカーを使用するようになり、出力トークンが削減された。API コスト削減とレスポンス速度の向上に寄与。
- **影響**: 日常的な編集操作が高速化し、トークン消費が抑えられる。

### --resume のセッション復元修正
- **ソース**: [GitHub CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)
- **要約**: v2.1.85 以前に作成されたセッションで `--resume` を使うと、deferred tools・MCP サーバー・カスタムエージェント利用時にプロンプトキャッシュが完全にミスする問題が修正された。
- **影響**: 長期セッションの再開が確実に動作するようになった。

### Jujutsu / Sapling VCS サポート
- **ソース**: [GitHub CHANGELOG.md](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

## Tips & Techniques（今週の新機能を活かす実践例）

### Tip 1: PreToolUse "defer" でヘッドレス承認ゲートを構築

本番デプロイ前に人間の承認を挟むフロー:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "if": "tool_input.command matches 'deploy|kubectl apply'",
        "command": "echo '{\"permissionDecision\": \"defer\", \"message\": \"Deploy requires human approval\"}'",
        "timeout": 5000
      }
    ]
  }
}
```

`--resume` で再開すると hook が再評価される。CI/CD での段階的デプロイ承認に活用可能。

- **ソース**: [Hooks Reference](https://code.claude.com/docs/en/hooks)

### Tip 2: PermissionDenied hook で auto mode 拒否をログ・リトライ

```json
{
  "hooks": {
    "PermissionDenied": [
      {
        "matcher": "",
        "command": "echo '{\"hookSpecificOutput\": {\"hookEventName\": \"PermissionDenied\", \"retry\": true}}' | tee -a /tmp/claude-denials.log",
        "timeout": 5000
      }
    ]
  }
}
```

auto mode で誤って拒否されたツールコールを自動リトライしつつログに記録。どのツールが頻繁に拒否されるか分析して allowRules を調整できる。

- **ソース**: [Hooks Reference](https://code.claude.com/docs/en/hooks)

### Tip 3: CLAUDE_CODE_NO_FLICKER でデモ録画品質向上

```bash
export CLAUDE_CODE_NO_FLICKER=1
claude
```

ターミナル録画ツール（asciinema 等）と組み合わせてクリーンなデモ動画を作成。社内プレゼンやブログ記事素材に。

- **ソース**: [ClaudeCodeLog on X](https://x.com/ClaudeCodeLog/status/2039153164717334706)

### Tip 4: Message Batches API で大規模コード生成

```python
import anthropic

client = anthropic.Anthropic()
batch = client.messages.batches.create(
    requests=[{
        "custom_id": "large-codegen",
        "params": {
            "model": "claude-opus-4-6",
            "max_tokens": 300000,
            "messages": [{"role": "user", "content": "Generate..."}],
        }
    }],
    betas=["output-300k-2026-03-24"]
)
```

リポジトリ全体のドキュメント生成やマイグレーションスクリプトの一括生成に活用。

- **ソース**: [Claude Platform Release Notes](https://platform.claude.com/docs/en/release-notes/overview)
