# Zellij 設定 — モバイルモード対応

## 設計方針

デスクトップは `Alt+大西配列` のチョード操作（最速）、モバイルは
**`Ctrl+a` プレフィックス → 単打キー**（tmux モード）に全操作を収束させる二層構成。
モバイルクライアントは Ctrl+a さえ送れれば、あとは修飾キーなしで完結する。

## tmux モード（Ctrl+a）キーマップ

| キー | 動作 | モード維持 |
|------|------|-----------|
| `t` / `s` | 前タブ / 次タブ | ✅ 連打可 |
| `n` / `r` | pane フォーカス 下 / 上（stack 内移動） | ✅ 連打可 |
| `1`–`9` | タブ番号ジャンプ | 抜ける |
| `z` | pane 全画面トグル | 抜ける |
| `b` | pane を新タブに昇格（BreakPane） | 抜ける |
| `f` | floating pane トグル | 抜ける |
| `Space` | レイアウト切替（base → **stack** → vertical → horizontal） | 抜ける |
| `S` / `N` | pane 分割 右 / 下（旧 s/n から格上げ） | 抜ける |
| `i` | スクロールモード | — |
| `l` | セッションモード | — |
| `q` | デタッチ | — |
| `Esc` / `Enter` | Normal に戻る | — |

ナビ系（t/n/r/s）はモードに留まるので、`Ctrl+a` → `s s s` でタブを3つ進められる。
打ち終わったら `Esc` か `Enter` で抜けてから通常入力する。

## モバイル（iPhone / Moshi）側の推奨設定

Moshi のショートカットビルダーでキーシーケンスをボタン / スワイプに割り当てる:

| 割当先 | シーケンス | 用途 |
|--------|-----------|------|
| ボタン or スワイプ | `Ctrl+a` `z` | 全画面トグル |
| ボタン | `Ctrl+a` `Space` | stack レイアウト切替 |
| ボタン | `Ctrl+a` `b` | pane→タブ昇格 |
| スワイプ左右 | `Ctrl+a` `t` / `Ctrl+a` `s` | タブ移動 |
| D-pad | `Ctrl+a` `n` / `Ctrl+a` `r` | stack 内 pane 移動 |

補助機能:

- **Sticky modifier**: Ctrl/Alt を1タップ→次のキーに適用、ダブルタップ→ロック。
  デスクトップ用の `Alt+文字` バインドもこれで一応打てる
- **長押し Ctrl**: Zellij / Claude Code 対応のショートカットパネルが出る
- **Mouse Mode**: タップが zellij に届くので compact-bar のタブ切替も可
- **iOS フローティングキーボード**: キーボードをピンチインで縮小し隅に退避できる

## レイアウト（layouts/mobile.kdl）

builtin compact 相当 + `stack`（完全スタック, min_panes=2）を先頭に追加したもの。
`Ctrl+a Space` 一発で「フォーカス pane だけ展開・他はタイトル1行」になり、
狭い画面でも読める。もう一度以降の Space で builtin 相当の分割系に循環する。

## 既知の制約

- 複数クライアント同時 attach 時は最小クライアントに合わせてリサイズされる
  （tmux の grouped session 相当は Zellij に無い）。スマホとデスクトップの
  同時利用時はスマホ専用セッションを分けること
- `zellij web`（0.44+）はスクロール閲覧には使えるが、iOS Safari のソフト
  キーボードに Ctrl が無いため prefix 操作の主力にはならない
