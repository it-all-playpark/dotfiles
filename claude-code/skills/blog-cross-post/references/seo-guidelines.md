# SEO Guidelines

Zenn/Qiitaクロスポスト時のSEO最適化ガイドライン。

## 重複コンテンツ対策

### Canonical URL

元記事を正規URLとして明示:

```markdown
<!-- 記事冒頭または末尾 -->
この記事は [playpark Blog](https://www.playpark.co.jp/blog/{slug}) からの転載です。
```

### 公開タイミング

1. **元記事を先に公開**（最低24-48時間前）
2. Googleがインデックス完了を確認
3. その後Zenn/Qiitaに投稿

## タイトル最適化

### Zenn向け

```
【Next.js 16】MDXブログを自作した話 - ISRと予約投稿で「ちょうどいい」を目指す
```

- 【】でカテゴリ/技術を明示
- キーワードを前方に配置
- 60文字以内推奨

### Qiita向け

```
【Next.js 16】MDXブログを自作した話 - ISRと予約投稿で「ちょうどいい」を目指す
```

- タイトルは同一でOK
- タグでキーワード補完

## キーワード戦略

### プラットフォーム別検索傾向

| プラットフォーム | ユーザー傾向 | キーワード例 |
|-----------------|-------------|-------------|
| Zenn | 最新技術、実装詳細 | Next.js 16, App Router, RSC |
| Qiita | 入門、ハウツー | 〇〇のやり方, 〇〇入門, 初心者 |

### seed情報の活用

seed/ディレクトリがある場合、以下を抽出してキーワード強化:

| seed file | 抽出情報 |
|-----------|---------|
| export.md | 技術スタック、主要機能 |
| context.md | 課題解決、ユースケース |
| pr-summary.md | 機能追加の変遷、改善点 |

## 内部リンク戦略

### 関連記事の選定基準

1. **投稿日**: 元記事の公開日以前の記事のみ（必須）
2. **同じカテゴリ**: 読者の興味関心が近い
3. **同じタグ**: 技術的な関連性
4. **同じseed**: 同プロジェクトの別記事

### リンクアンカーテキスト

```markdown
<!-- Good: 具体的なキーワード -->
[Next.jsでISRを実装する方法](https://...)

<!-- Bad: 汎用的 -->
[こちら](https://...)
```

## 構造化データ対策

### 見出し構造

```markdown
# タイトル (H1 - 1つのみ)

## この記事で学べること (H2)

## セクション1 (H2)

### サブセクション (H3)

## まとめ (H2)

## 元記事・関連情報 (H2)
```

### 箇条書きの活用

```markdown
## この記事で学べること

- MDXファイルベースのコンテンツ管理
- ISRで「ビルドしなくても更新される」を実現
- JST対応の予約投稿
```

検索結果でリッチスニペット表示の可能性向上。

## プラットフォーム固有のSEO

### Zenn

- **Topics**: 検索に大きく影響、適切な5つを選択
- **Emoji**: 目立つ絵文字で CTR 向上
- **Publication**: 企業・チームなら Publication 活用

### Qiita

- **Tags**: バージョン情報付きで検索精度向上
- **LGTM/ストック**: 初期の反応が検索順位に影響
- **限定共有 → 公開**: 下書き期間で内容精査

## コンテンツ品質チェックリスト

### 転載前確認

- [ ] タイトルにメインキーワード含む
- [ ] H2見出しにサブキーワード含む
- [ ] 導入文で記事の価値を明示
- [ ] コード例が動作確認済み
- [ ] 画像に alt テキストあり
- [ ] 内部リンク 2-3 個含む
- [ ] CTA セクションあり

### 転載後確認

- [ ] canonical URL 明記済み
- [ ] 元記事への導線あり
- [ ] プラットフォーム固有フォーマット適用済み
- [ ] タグ/トピックス適切に設定

## 分析・改善

### トラッキング

元記事のGAでリファラー分析:
- Zenn/Qiitaからの流入数
- 直帰率の比較
- コンバージョン（お問い合わせ等）

### 改善サイクル

1. 記事公開
2. 2週間後に流入分析
3. 効果の高いパターンを特定
4. 次回記事に反映
