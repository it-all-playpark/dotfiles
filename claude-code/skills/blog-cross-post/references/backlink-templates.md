# Platform-Differentiated Templates

ZennとQiitaで**切り口を変えた**テンプレート。同一内容の重複を避け、各プラットフォームのユーザー特性に最適化。

## プラットフォーム特性

| Platform | 読者層 | 好む切り口 | 記事スタイル |
|----------|--------|-----------|-------------|
| **Zenn** | モダン技術志向、実装重視 | **How（実装パターン）** | コード先行、実践的、すぐ使える |
| **Qiita** | 幅広い技術者、背景理解重視 | **Why（技術選定理由）** | 解説先行、比較検討、判断材料 |

## 重要ルール

### 1. 対象カテゴリの確認

| Category | 対応 |
|----------|------|
| tech-tips | このテンプレートを使用 |
| lab-reports | このテンプレートを使用 |
| solutions | エラー終了 |
| case-studies | エラー終了 |

### 2. 差別化の原則

**NG: 同一内容**
- ZennとQiitaで構成・切り口が同じ
- SEO効果が分散、読者にも価値なし

**OK: 切り口が異なる**
- Zenn: 「こう実装する」から始まる
- Qiita: 「なぜこうするか」から始まる
- 同じ技術トピックでも読後感が異なる

### 3. 元記事から抽出する情報

| 変数 | 抽出元 | 説明 |
|------|--------|------|
| `{CORE_TOPIC}` | 記事の主要テーマ | コア版で扱う範囲 |
| `{CODE_EXAMPLE}` | 元記事のコード | **動作する完全なコード** |
| `{WHY_THIS_APPROACH}` | 元記事の背景・理由 | なぜこのアプローチを選んだか |
| `{ALTERNATIVES}` | 元記事の比較・検討 | 他の選択肢との比較 |
| `{DEPTH_1}` | 元記事の応用セクション | 公式版のみの内容 |
| `{DEPTH_2}` | 元記事の知見セクション | 公式版のみの内容 |
| `{DEPTH_3}` | 元記事の比較・検証 | 公式版のみの内容 |
| `{ORIGINAL_URL}` | Init結果 | `original_url` |
| `{ARTICLE_TITLE}` | 元記事frontmatter | `title` |

## 深掘り導線の書き方ガイド

### プラットフォーム別の導線ニュアンス

| Platform | 導線のトーン | 例 |
|----------|-------------|-----|
| Zenn | 「さらに実践的な内容」 | 「複数パターンの実装例」「パフォーマンス最適化」 |
| Qiita | 「さらに詳しい解説」 | 「実運用での知見」「失敗パターンと対処法」 |

### 具体的に書く

**NG: 抽象的**
```markdown
- 詳細な実装
- 応用例
- その他の知見
```

**OK: 具体的**
```markdown
# Zenn版（How寄り）
- 🔄 ISR/SSG/SSRそれぞれの実装パターン
- ⚠️ キャッシュ制御のエッジケース対応
- 📊 100記事規模でのビルド時間ベンチマーク

# Qiita版（Why寄り）
- 💭 ISR vs SSG vs SSRをどう選ぶか
- ⚠️ 本番運用で遭遇した問題と判断
- 📋 チーム導入時の検討ポイント
```

## 文字数ガイドライン

| セクション | Zenn | Qiita |
|-----------|------|-------|
| TL;DR/学べること | 100-150字 | 100-150字 |
| 背景・課題 | 100-150字 | 200-300字 |
| 本文（コード・解説） | 800-1200字 | 600-1000字 |
| まとめ | 100-150字 | 100-150字 |
| 深掘り導線 | 150-200字 | 150-200字 |
| **合計** | **1400-2200字** | **1200-2000字** |

**Zennはコード多め、Qiitaは解説多め**になる傾向。
